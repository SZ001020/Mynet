# 长篇学术论文 Markdown 翻译技能指南 (Paper Translation Skill)

## 核心目标
在已有的学术论文 Markdown 文件基础上翻译出新的论文 md 文件。要求：
- **精准汉化**：正文、图注（Caption）、表注等内容均翻译为流畅专业的中文。
- **格式保真**：必须 100% 保持原始图片链接 (`![](...)`)、表格结构 (`<table>...</table>`) 及其他 HTML 标签原样不动。
- **公式免疫**：行内公式（`$...$`）和区块公式（`$$...$$`）直接沿用原版，**绝不干预或改动**。
- **特定跳过**：参考文献（References）部分无需翻译，直接保留英文原文。

## 标准执行工作流 (Workflow)

由于长篇论文可能会超出单次 AI 对话的上下文限制，并容易导致格式截断，请严格按照以下**分步策略**执行：

### Step 1: 制定框架与大纲
1. 首先通读（或使用工具检索）原 md 文件的脉络。
2. 生成一份结构大纲文件（如 `full_translated_structure.md`），指出需要翻译的各个章节标题，明确“参考文献不翻译”等原则。
3. 同用户确认翻译架构后，再开始正式动工。

### Step 2: 分区分块提取文本
1. 不要尝试一次性读取或生成整篇文档。
2. 使用终端工具（如 `sed -n '/^# Section I/,/^# Section II/p' full.md`）按章节（如 Abstract, Part 1, Part 2...）精确切分和提取原版英文文本内容。
3. 如果内容输出过长，请利用临时文件流转，再通过读取临时文件来获取英文原文。

### Step 3: 实施翻译并脚本化写入 (关键避坑点)
1. **严格使用 Python 脚本写入**：由于 Markdown 含有大量 LaTeX 特殊符号（如 `\`、`$$`）及复杂排版，如果在终端使用 `cat << 'EOF'` 追加文本，极易发生转义错误或解析失败。
2. **构建追加脚本**：将每一部分的翻译结果，使用 Python 脚本通过原样字符串（`r""" 这里放翻译结果 """`）的方式精准追加（append）或覆盖到目标产出文件（如 `translated_paper.md`）中。
3. **查缺补漏**：在写入前验证所有的图片链接、表格 HTML 及公式是否跟提取的原文一模一样无损。

### Step 4: 循环推进与收尾
1. 重复 Step 2 与 Step 3，以“提取一段 -> 翻译一段 -> 用 Python 脚本安全追加一段”的模式流水线作业。
2. 直到完成所有前文核心章节。
3. 对于最后的 References 章节，直接写 Python 程序将原文件的对应尾部内容提取并直接 `append` 到翻译文件的最末尾。

## 提示词模板 (Prompt Template 用于初始唤醒)
> "请作为专业学术翻译助手，按照我给定的论文 Markdown 文件，遵循『结构化切分翻译、正文图注汉化、公式图片表格原样保留、参考文献不翻译』的原则，逐步采用 Python Raw 脚本写入法，帮我把整篇文献翻译为中文。首先请帮我提取并翻译第一部分的标题及大纲即可。"
