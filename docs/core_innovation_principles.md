# 核心创新原理（第三版，结合 v1.1 与参考论文）

## 1. 重新界定问题
结合 MFNet、FTransUNet、ASMFNet、UNetFormer、RSPrompter 和 APSAM 这些参考论文，可以提炼出一个更清楚的判断：
- 论文主张必须收束，不能把所有想到的模块并列成“创新点”。
- 每个模块都要能对应一种直接可观测的实验现象。
- 真正有说服力的稿件，不只靠主结果表，还靠消融、效率、可视化和稳定性一起闭环。

据此，本项目不应再被定义为“在 MFNet 上继续加模块看能否涨点”，而应定义为：

在多模态遥感语义分割中，MFNet 已经证明了 RGB 与 DSM 联合建模以及 SAM 参数高效微调是有效的，但在由 same-domain 走向 weak-cross-domain 时，仍然存在三类关键退化：
- 语义特征域偏移，导致 target 域整体判别能力下降。
- 道路、建筑等对象的边界与完整性退化，尤其在 target 域更明显。
- 若进一步依赖伪标签驱动训练，固定阈值筛选容易把噪声直接传入结构损失与后续优化。

因此，本工作的目标不是再设计一个更重的主干网络，而是基于 MFNet 构建一个面向弱跨域场景的统一训练框架，在尽量保持参数高效微调优势的前提下，同时提升：
- target 域语义一致性；
- target 域结构完整性；
- 伪标签与提示驱动训练的稳定性。

## 2. 参考论文给出的启发与边界

### 2.1 来自 MFNet 的启发
MFNet 的核心价值不只是性能，更重要的是它说明了两件事：
- SAM 底座可以被有效迁移到多模态遥感分割；
- LoRA 和 Adapter 这类参数高效微调机制可以形成实际可用的性能-成本平衡。

这意味着当前项目最合理的起点仍然是 MFNet，而不是重新发明一套全新的大 backbone。

### 2.2 来自 FTransUNet 和 ASMFNet 的启发
FTransUNet 与 ASMFNet 强调的不是“模块越多越好”，而是：
- 多层或多尺度信息确实重要；
- 但这些信息必须通过可以被验证的结构分析或消融来证明有效；
- 更复杂的融合结构会带来额外复杂度，因此必须有明确收益。

对当前项目的启发是：
- 多层域对齐值得尝试；
- 但它应当是 high-level UDA 稳定之后的条件增强，而不是第一天就默认启用。

### 2.3 来自 UNetFormer 的启发
UNetFormer 的实验部分非常值得借鉴，因为它同时强调：
- 精度；
- 模块必要性；
- 效率；
- 稳定性与结构合理性。

因此，本项目的正式文档和实验设计，必须从一开始就预留效率分析、消融分析和可视化证据，而不是等主结果跑完后再补。

### 2.4 来自 RSPrompter 和 APSAM 的启发
RSPrompter 与 APSAM 共同说明了一点：
- 在遥感场景里，SAM 的效果对 prompt 的类型、位置和质量高度敏感；
- 自动提示可以改善边界质量与伪标签质量；
- 但 prompt 机制本身并不能自动迁移成“弱跨域语义分割主创新”，因为它们对应的任务分别更偏实例分割和弱监督提取。

因此，对当前项目更稳妥的定位是：
- prompt-guided 机制可以作为 v1.1 的条件增强；
- 它更适合被写成伪标签精炼与稳定性增强分支；
- 只有在实验上形成稳定增益后，才应该进入论文主贡献，否则应降级到附加实验或附录。

## 3. 当前建议采用的技术路线

### 3.1 主线必须先收束
结合当前仓库条件与文献经验，正式主线建议分成三层：

1. 必做主线 M0
	以 MFNet 为底座，采用单层 high-level 域对齐，加边界与对象结构约束，优先解决 weak-cross-domain 下最直接的语义偏移与结构退化问题。

2. 条件增强 M1
	在 M0 已稳定的前提下，再扩展到 mid+high 的多层域对齐，用于验证中层特征是否能进一步改善小目标与细边界。

3. 条件增强 M2
	在 M0 或 M1 已稳定的前提下，引入 prompt-guided 伪标签精炼，将自动点提示、框提示或混合提示作为伪标签质量提升手段，而不是一开始就作为主理论核心。

### 3.2 为什么这样分层
这样分层的目的是避免三种常见失败：
- 同时上线太多机制，最后不知道谁有效；
- 基础 UDA 尚未稳定时就叠 prompt 分支，导致训练解释失效；
- prompt 分支失败后拖累整篇论文叙事。

### 3.3 第 6 周起的执行转向
结合当前实验结果波动与证据缺口，执行策略在第 6 周起调整为：
- 先同域，再跨域；
- 先机制闭环，再迁移验证；
- 先稳态统计，再追求峰值。

具体要求：
- 第 6 周优先完成 same-domain 的多 seed 机制证据（边界、对象、模态、微调策略）。
- 第 7 周仅回灌第 6 周通过门槛的模块到 weak-cross-domain。
- 第 8 周再做封板复验与最终主叙事收束。

## 4. 数学表达与版本化损失设计
设 source 样本为 $(x_s, y_s)$，target 样本为 $x_t$。模型输出分割 logits 为 $p$，分层特征为 $\{f_l\}$，高置信掩码为 $M_\tau$。

same-domain 阶段建议使用：

$$
\mathcal{L}_{same} = \mathcal{L}_{seg}^{src} + \lambda_{bdy}\mathcal{L}_{bdy}(p_t, M_\tau) + \lambda_{obj}\mathcal{L}_{obj}(p_t, M_\tau) + \lambda_{prm}\mathcal{L}_{prompt}(p_t, M_\tau)
$$

weak-cross-domain 阶段建议使用统一形式：

$$
\mathcal{L}_{weak} = \mathcal{L}_{seg}^{src} + \lambda_{adv}\sum_{l \in \mathcal{S}}\mathcal{L}_{adv}^{(l)} + \lambda_{bdy}\mathcal{L}_{bdy}(p_t, M_\tau) + \lambda_{obj}\mathcal{L}_{obj}(p_t, M_\tau) + \lambda_{prm}\mathcal{L}_{prompt}(p_t, M_\tau)
$$

其中：
- $\mathcal{L}_{seg}^{src}$：source 域监督分割损失，是整个训练的主锚点。
- $\mathcal{L}_{adv}^{(l)}$：第 $l$ 层域对齐损失，用于减小 source-target 语义分布差异。
- $\mathcal{L}_{bdy}$：边界约束，用于减轻 target 域边界断裂与毛刺。
- $\mathcal{L}_{obj}$：对象一致性约束，用于增强建筑、道路等对象的区域完整性。
- $\mathcal{L}_{prompt}$：prompt-guided 一致性或精炼损失，用于降低固定阈值伪标签的噪声传播。

建议默认启动方式：
- 第一阶段：$\mathcal{S}=\{high\}$，$\lambda_{prm}=0$。
- 第二阶段：若 UDA 稳定，再尝试 $\mathcal{S}=\{mid, high\}$。
- 第三阶段：若伪标签质量与训练稳定性允许，再启用 $\lambda_{prm}>0$ 的 prompt 分支。
- 第 6 周起执行优先级：先在 same-domain 验证上述阶段，再回灌到 weak-cross-domain。

这里最关键的是版本化启用顺序，而不是一开始就把所有项写进总损失。

## 5. 为什么这条路线成立

### 5.1 域对齐处理全局语义偏移
high-level 特征对纹理变化更稳，是 weak-cross-domain 最先应对的偏移来源。若再加入中层特征，可补足局部边界和小目标信息，但前提是训练不失稳。

### 5.2 结构约束补足对抗训练短板
只做域对齐往往会牺牲边界质量。边界与对象损失提供局部几何先验，更容易在道路、建筑等边界敏感类别上体现价值。

### 5.3 多模态信息提供稳定锚点
DSM 之类的几何模态在跨域时通常比纯纹理更稳定，因此它不只是“再多一个通道”，而是 target 域结构泛化的重要支撑。

### 5.4 参数高效微调降低迁移成本
LoRA 和 Adapter 的作用不只是省参数，更是降低过拟合和部署成本，这一点在论文里应作为效率卖点被保留。

### 5.5 prompt-guided 分支的真实价值是“伪标签精炼”
参考 APSAM 和 RSPrompter，prompt 的主要贡献不应被表述为“又多了一个 SAM 模块”，而应表述为：
- 用点/框提示把粗糙高置信区域转成更结构化的伪监督；
- 降低固定阈值伪标签对边界的破坏；
- 改善 early stage 训练波动。

但这条论点只有在实验中成立时才应进入论文主线。

## 6. 论文贡献建议写法

### 6.1 安全版贡献写法
如果最终稳定有效的是主线 M0 或 M1，建议论文只保留以下 3 到 4 个贡献：

1. 提出一个基于 MFNet 的多模态弱跨域遥感分割统一训练框架，在参数高效微调底座上联合建模域不变性与结构一致性。
2. 提出一种面向 target 高置信区域的结构约束策略，通过边界与对象级一致性缓解弱跨域场景中的边缘破碎与对象不完整问题。
3. 设计一套由 same-domain 机制验证到 weak-cross-domain 主结论验证的分阶段实验协议，使方法增益可以被逐层解释。
4. 通过主结果、模块消融、模态分析、效率分析和可视化对比，系统证明方法在 target 域精度、边界质量和参数效率上的综合优势。

### 6.2 增强版贡献写法
如果 prompt-guided 分支确实稳定有效，可以把第二条或第四条扩展为：

提出一种面向 target 高置信区域的 prompt-guided 伪标签精炼机制，通过自动提示与结构一致性联合约束降低伪标签噪声并改善 target 域边界质量。

但若这条分支不稳定，不应强行写入标题和摘要主贡献。

## 7. 论文主张对应的证据链

### 7.1 若声称“整体方法有效”
必须提供：
- target 域主结果对比表；
- 至少 2 个数据协议下的结果；
- 3 个随机种子的均值和方差。

执行顺序补充：
- 第 6 周先完成 same-domain 侧的机制证据和多 seed 统计；
- 第 7 周再提交 weak-cross-domain 的回灌结果与稳定性统计。

### 7.2 若声称“结构约束改善边界与对象完整性”
必须提供：
- 去除边界损失和对象损失的消融表；
- 道路、建筑等类别的单类 F1 或 IoU 增益；
- 至少 2 组定点放大可视化图。

### 7.3 若声称“多层域对齐是必要的”
必须提供：
- single-level 与 multi-level 对齐对比表；
- 中层对齐对小目标、边界或某些类别的具体收益；
- 训练稳定性对比，而不是只比 best score。

### 7.4 若声称“prompt-guided 分支有效”
必须提供：
- 固定阈值伪标签与 prompt-guided 伪标签的对比表；
- 有效伪标签像素占比、类别占比或边界质量统计；
- 至少 1 组置信图、伪标签图或局部放大图；
- 至少 1 组稳定性证据，例如 early stage 振荡减弱。

### 7.5 若声称“多模态和参数高效微调是必要的”
必须提供：
- RGB-only 与 RGB+DSM 对比表；
- LoRA、Adapter、冻结或全量微调对比表；
- 可训练参数量、显存占用、推理时间或吞吐对比。

## 8. 实验实施证据规范
为保证后续写论文时不缺材料，训练阶段必须固定保存以下证据。

### 8.1 每次实验必须保存的数据
- 运行配置快照：yaml、命令行参数、git commit、随机种子、环境版本。
- 训练曲线原始数据：iter 级和 epoch 级 loss 与 metric。
- 伪标签统计：高置信像素占比、类别分布、阈值命中率。
- 域对齐统计：域判别器准确率、对抗损失曲线、各层损失明细。
- prompt 统计：若启用 prompt，必须保存提示数量、类型、有效区域比例。
- 模型状态：best、last、optimizer、scheduler。
- 验证与测试明细：source/target 全量指标 json 和每类 IoU/F1。

### 8.2 论文阶段必须具备的图片
- 方法总览图。
- train/val 损失与指标曲线。
- same-domain 与 weak-cross-domain 可视化对比图。
- 边界局部放大图。
- 伪标签质量图、热图或特征分布图至少一种。
- failure cases 至少 3 组。

### 8.3 论文阶段必须具备的表格
- 主结果表：same-domain、weak-cross-domain、optional stress。
- 组件消融表：至少包含 -adv、-bdy、-obj、-DSM。
- 多层对齐分析表：single-level 与 multi-level。
- 模态与微调策略分析表：RGB、RGB+DSM、LoRA、Adapter。
- 效率分析表：参数量、可训练参数、显存、吞吐或推理时间。
- 稳定性或数据效率表：至少一项。
- 若 prompt 分支保留：额外增加 prompt 消融表。

## 9. 最低可投稿门槛与降级规则

### 9.1 最低门槛
- 主结论不能只建立在 same-domain。
- weak-cross-domain 需要稳定正增益。
- 不能只报单次最优值。

比较稳妥的门槛是：
- 相对 source-only 或纯监督迁移基线，target mIoU 平均提升约 1.0 到 1.5；或
- 提升不到 1.0，但同时具备明显边界质量优势、低方差和更优效率指标。

### 9.2 降级规则
- 若 multi-level 对齐不稳定，则退回 high-level 对齐作为正式主模型。
- 若 prompt 分支没有稳定正增益，则保留为可选分析或附录，不进入标题、摘要和主贡献。
- 若 same-domain 结构损失不涨点，但 weak-cross-domain 边界质量确实改善，仍可保留其作为跨域结构修复模块。
- 若第 6 周 same-domain 仍无法给出清晰机制证据，则第 7 周暂停引入新跨域模块，优先完成机制闭环。

## 10. 明确不建议的路线
- 不建议再额外设计一个大而复杂的新主干网络。
- 不建议把多层对齐、结构损失和 prompt 分支同时默认上线。
- 不建议只在一个数据集上证明主结论。
- 不建议只报 overall mIoU，不分析类别和边界。
- 不建议 prompt 分支尚未稳定时就在文档里把它写成“核心创新”。

## 11. 当前版本的正式定位
基于当前仓库状态与参考论文，更现实也更稳妥的论文定位是：

以 MFNet 为底座，提出一个面向弱跨域遥感语义分割的统一训练框架，并通过结构化实验说明：
- 多模态参数高效微调提供稳定语义锚点；
- 轻量域对齐改善 target 域整体判别；
- 边界与对象一致性约束修复 target 域结构退化；
- 多层域对齐和 prompt-guided 精炼是条件增强，而不是默认核心。

并在执行层面采用“第 6 周起先同域后跨域”的策略：
- 第 6 周聚焦 same-domain 机制闭环和稳态统计；
- 第 7 周只回灌通过门槛的机制到 weak-cross-domain；
- 第 8 周完成封板复验与主表主图定稿。

这样既吸收了 v1.1 中更积极的扩展想法，也保留了当前正式版最重要的优点：主张清晰、实现可控、结果可解释。
