# SSRS 统一跨域多模态分割项目：实验排期表（按周与 GPU 天）

## 1. 目标与约束
- 目标：在现有 SSRS 代码基础上完成第一版统一框架（MFNet + UDA + 边界约束），输出可投稿级实验结果。
- 主要数据场景：ISPRS Vaihingen/Potsdam（主线），先做同域与弱跨域，再做跨域增强。
- LoveDA 定位：可选压力测试，不作为主线验收门槛。
- GPU 预算口径：1 GPU-day = 1 张 GPU 连续训练 24 小时。

## 2. 设备假设
- 训练卡：1 到 2 张 24GB 显存级别 GPU（例如 RTX 4090/A5000/A6000 同级）。
- 周可用时长：7x24 小时持续运行。
- 若你只有 1 张卡，整体周期约增加 20% 到 35%。

## 3. 周排期总览（8 周）

| 周次 | 里程碑 | 关键任务 | 预计 GPU-day |
|---|---|---|---:|
| 第 1 周 | 基线可复现 | 跑通 MFNet 基线与日志模板，固定随机种子与评估脚本 | 2.0 |
| 第 2 周 | 同域增强 | 接入边界/对象约束，先在 Vaihingen 同域验证收益 | 2.5 |
| 第 3 周 | 弱跨域原型 | Vaihingen->Potsdam 或 Potsdam->Vaihingen 迁移原型 | 3.0 |
| 第 4 周 | UDA 稳定化 | 域对齐权重、判别器学习率、warmup 网格搜索 | 3.5 |
| 第 5 周 | 结构联合训练 | UDA + 边界对象约束联合训练与可视化 | 3.0 |
| 第 6 周 | 消融实验 | 去 UDA/去边界/去多模态/微调策略对照 | 4.0 |
| 第 7 周 | 参数与数据效率 | LoRA/Adapter/全量微调 + 10%/30%/100% 标注比例 | 3.0 |
| 第 8 周 | 整理与复验 | 最优方案复训、统计显著性、表格定稿 | 1.5 |

总计建议预算：22.5 GPU-day
可选 LoveDA 压力测试追加预算：4.0 到 6.0 GPU-day

## 4. 每周细分执行清单

### 第 1 周：基线冻结周（2.0 GPU-day）
- 训练与评估统一
  - 统一窗口大小、stride、batch、学习率策略。
  - 统一评估指标：mIoU、F1、Boundary F1。
- 产出
  - 基线结果表 v1（MFNet、FTransUNet、ASMFNet 各 1 次跑通）。
  - 标准日志模板（csv + 可视化图）。
- GPU 分配
  - MFNet：1.0
  - FTransUNet/ASMFNet 对照：1.0

### 第 2 周：同域增强验证（2.5 GPU-day）
- 工作项
  - 接入 SAM_RS 的 boundary/object loss。
  - 在同域设置验证结构约束收益与训练稳定性。
- 实验
  - Vaihingen 同域 1 组完整训练。
  - 与无结构约束的 MFNet 对比。
- GPU 分配
  - 结构约束训练：1.8
  - 对照与复跑：0.7

### 第 3 周：弱跨域最小可行版（3.0 GPU-day）
- 工作项
  - 在 MFNet 高层语义特征处新增域判别分支。
  - 引入对抗损失，仅做 feature-level 对齐。
- 产出
  - Vaihingen->Potsdam 或反向的第一版迁移结果。
- GPU 分配
  - 原型训练：2.0
  - 对照与复跑：1.0

### 第 4 周：UDA 收敛与稳定性（3.5 GPU-day）
- 工作项
  - 调整对抗损失权重、判别器学习率、warmup 轮数。
  - 三随机种子复跑。
- 产出
  - 收敛曲线与稳定性报告。
- GPU 分配
  - 网格搜索：2.3
  - 三种子：1.2

### 第 5 周：结构联合训练（3.0 GPU-day）
- 工作项
  - 在弱跨域训练中加入 boundary/object 结构约束。
  - 在高置信伪标签区域施加结构损失。
- 产出
  - 边界质量显著可视化（道路/建筑边界）。
- GPU 分配
  - 联合训练：2.2
  - 可视化与复验：0.8

### 第 6 周：核心消融（4.0 GPU-day）
- 消融矩阵
  - -UDA
  - -Boundary/Object
  - -Multimodal（只 RGB）
  - LoRA 与 Adapter 切换
- GPU 分配
  - 4 条主消融：3.6
  - 补跑：0.4

### 第 7 周：参数效率与数据效率（3.0 GPU-day）
- 工作项
  - 比较全量微调、LoRA、Adapter 的精度与参数量。
  - 标注比例 10%/30%/100% 数据效率对照。
- 产出
  - 精度-参数效率图与数据效率曲线。
- GPU 分配
  - 三策略训练：2.5
  - 补跑：0.5

### 第 8 周：复验封板（1.5 GPU-day）
- 工作项
  - 最优模型复训一次。
  - 导出全部表格与可视化图。
- 产出
  - 论文主表、消融表、可视化图、统计区间。
- GPU 分配
  - 复训：1.0
  - final test：0.5

## 5. 风险缓冲（建议预留）
- 额外预留：3.0 GPU-day
- 触发条件
  - 对抗训练振荡。
  - 伪标签噪声导致指标回退。
  - 随机种子方差过大。

## 6. 验收标准（每阶段）
- 阶段 A（第 1-2 周）
  - MFNet 基线可复现，误差不超过历史结果的 1.0 mIoU。
- 阶段 B（第 3-4 周）
  - 弱跨域 UDA 至少带来 1.0 mIoU 提升。
- 阶段 C（第 5-8 周）
  - UDA + 结构约束相对弱跨域基线再提升 0.5 到 1.0 mIoU。
  - 完成完整消融矩阵，主结论在 3 个种子下方向一致。

## 8. 可选附加阶段（LoveDA 压力测试）
- 建议在主线完成后追加 1 到 2 周。
- 目标
  - 检验方法在强域偏移场景的鲁棒性，而非追求绝对最高分。
- 结果呈现
  - 重点报告相对提升、边界质量与稳定性，不以绝对 mIoU 作为唯一结论。

## 7. 建议日常节奏（执行层）
- 周一到周二：主训练与对照启动。
- 周三：中期指标检查，必要时提前止损并改参数。
- 周四到周五：补跑和可视化。
- 周末：整理日志、更新结果表、下周任务预排。
